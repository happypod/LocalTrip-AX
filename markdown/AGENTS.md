# AGENTS.md

## Project Name

LocalTrip AX / 소원로컬트립 MVP

## Project Purpose

본 프로젝트는 소원권역을 시작으로 숙박, 로컬체험, 주민소득사업, 로컬상품, 교육·인증, 예약문의, 성과관리를 연결하는 지역관광 AX 플랫폼의 MVP를 구축하는 것을 목적으로 한다.

MVP의 핵심은 완성형 OTA 예약 플랫폼이 아니라, 다음 기능을 안정적으로 구현하는 것이다.

1. 소원권역 숙박·체험·주민소득상품을 사용자에게 보여준다.
2. 사용자가 전화, 카카오, 네이버예약, 문의폼을 통해 사업자와 연결된다.
3. 관리자 또는 지역운영자가 숙소·체험·코스·주민사업체 정보를 등록하고 관리한다.
4. 조회, 클릭, 문의 데이터를 수집하여 향후 AX 운영진단의 기반 데이터를 만든다.
5. 주민사업체 교육·인증·보수교육 기능은 MVP에서는 기본 구조만 만든다.
6. 결제, 실시간 예약, 자동 정산, 차량 운송, 후기 시스템은 MVP에서 제외한다.

## Core Concept

이 플랫폼은 단순 숙박예약 사이트가 아니다.

정의:

> 주민이 운영하는 숙박·체험·로컬상품을 디지털 상품으로 전환하고, 예약문의·교육·인증·성과관리를 지원하는 지역관광 운영 플랫폼

MVP 실증 브랜드:

> 소원로컬트립

장기 플랫폼명:

> LocalTrip AX / 로컬트립 AX

## MVP Service Positioning

- B2C: 관광객이 숙소, 체험, 코스, 로컬상품을 탐색하고 문의한다.
- B2B: 숙소주, 체험운영자, 주민사업체가 입점하고 상품을 관리한다.
- B2G/B2R: 지자체, 앵커조직, 주민협의체, 협동조합이 성과관리와 주민소득사업 운영에 활용한다.

## Implementation Principle

1. MVP는 작게 만든다.
2. 데이터 구조는 처음부터 다지역 확장형으로 설계한다.
3. 모든 주요 테이블에는 regionId를 포함한다.
4. 숙소, 체험, 코스, 주민사업체, 문의, 클릭 이벤트를 분리한다.
5. 결제, 정산, 실시간 객실 재고는 넣지 않는다.
6. 운송 서비스는 직접 구현하지 않는다.
7. 모든 상품 상세에는 안전안내, 운영주체, 문의방식이 들어가야 한다.
8. 관리자 CRUD는 단순하지만 안정적으로 만든다.
9. 모바일 우선 UI를 적용한다.
10. 공공사업 보고와 AX 분석을 위해 LeadEvent 데이터를 반드시 수집한다.

## Forbidden in MVP

MVP에서 구현하지 않는다.

- 자체 결제
- 자동 정산
- 실시간 객실 재고
- 숙박 예약 확정 기능
- 운송 배차
- 차량 호출
- 후기 시스템
- 포인트/쿠폰
- 앱 출시
- 복잡한 AI 추천
- 다국어 자동 번역
- 자동 가격 추천
